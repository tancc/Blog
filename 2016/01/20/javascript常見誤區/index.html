Javascript常見誤區

有些javascript的語言特性是一些初學課程不會提到的，而很有可能在之後的程式中就因此發生了bug而不自知，
這裡就幾個部分來精簡說明

  • Pass by What

  • Equality

  • Function Scope

  • Hoisting

  • This

    這裡都講得滿粗淺的，目的是要快速瞭解大概，因此講的不嚴謹可能有說錯或誤解的很嚴重的地方，請大家在指證我^^

Pass by What

先講結論

  • 都是Pass by value
  • 當型態為Object，此時的value為reference

這裡的Object不單純為使用大括號建構的東西，其實Array, Function都算是Object。而value 為 reference的時候，其實傳遞是某個東西的位址，並沒有去複製一份出來，反之則有。

Equality

進行”是否相等”的比較時，通常會用到==或是===，一般的分法是”前者比較值是否相等“而”後者比較值以及型態是否相等“，大致上這麼解讀是沒錯的
，但並不完全正確。前者是比較”經過強制型別轉換(coercion)後的值是否相等“，後者是比較”未強制型別轉換(coercion)的值是否相等“。

進行a == b時，JS會先進行一系列的型態轉換，型態一致後再去比對value，因此會得到true。此外最後是轉變成1 == 1，反之後者由於沒有coercion
因此會得到false。
值得注意的地方在於，比較array時

第一個比對會進行coercion將兩邊都轉成字串，因此得到true。而第二個比對會false的原因在於，這是兩個object的比較，表面上看起來值都是一樣的，但是reference並不一樣因此會得到false。

Function Scope

這裡只介紹function-scope的概念，不對scope這個大議題作解釋，先講結論：

  • JS為function scope，也就是只有function才會創造出scope，其他像for, if, else這些有大括號{}都沒有產生出新的scope(es6才能用block-scope)

這裡可以看到，在function內宣告的變數，只有function內知道它的存在，也就是區域變數，這點很直覺。但在if 或for的{}內宣告的變數會都變成全域變數！

This

在執行function的時候，現在的this到底是什麼？這是一開始滿常會遇到的問題。這裡很敘述簡單的判斷方法，先講結論

  • this的內容跟function怎麼被呼叫有關
  • 四種invocation，直接呼叫、當作object的方法、用new、使用bind(),apply()或call()
