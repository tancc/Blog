react 處理非同步action的對策

在開發時，時常會有需要發一個action，然後去向後端拿資料的動作。
而這整個請求資料的行為是非同步的，因此在執行時還需要處理一些UI上的互動，如“載入中”、“發生錯誤”、“順利取得資料”。

這篇文分使用的範例應用為，輸入一個group的名稱，然會取得其成員的列表

如此可見，在component中需要去處理很多事情，這只是處理一個async action的狀況，實際情形會有非常多個或是有順序性關係，而這時候可以想像component程式碼會變得非常冗長。除此之外，component因此需要許多state去對應相關的情況，且變成impure。

Reducers 部分

首先先定義我們的reducers - index.js。步驟我自己是大致分為這漾：

  1. 思考整個state tree的架構。
  2. 思考這些tree中個別的state，要怎麼經由reducer來改變他得值。

1. 思考整個state tree的架構。

  • 我需要知道我現在所得到的資料是哪個group的。
  • 我有3個group，所以應該要有地方去存他們個別的member資料。
  • 每個group中需要記錄他們的資料(dataList)、資料總數(total)、資料位移量(offset，用於下次跟後端請求資料時的參數)、是否還有資料可以載入(end)，以及此項目資料是否正在載入中(loading)和是否有錯誤(errorMessage)。

寫起來很落落長，但是每個reducer就都變得很單純，只專注於處理一個state，哪裡出錯了都是很好追蹤的。
