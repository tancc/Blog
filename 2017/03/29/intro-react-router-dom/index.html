使用react-router-dom

使用react-router-dom

只記錄比較常用到的，詳細的內容可以參照官方文件

安裝

npm install react-router-dom@next -S or yarn add react-router-dom@next

介紹到的API

這篇沒有講到所有的API，只是大略簡單記錄一下

< Router >

V4版以前，都會在最外層用<Router>...</Router>將內部的<Route />包裹起來

在新版中會通常會以<BrowserRouter>來代替

此外還有其他許多Router，可以參照官方文件...

< Route >

<Route>完全就只是個component，他根據props來判斷要不要render出對應的component。

因為他是個component，所以可以做各式各樣的組裝。

這裡介紹他所使用的一些props：

  • path的規則更活了

支援更多的正則表達式，以下是部份範例。

  • exact

只有當url與path完整match條件才成立

  • 三種render methods(component, render, children)

component要傳入的是React component，而後兩者則是function

要注意的是當path相同時優先順序為component > render > Children

其中用children的話，不管path有沒有match他都會render

  • <Route>會自動傳入的props有history, location, match

< Link >

  • to傳入物件(舊版本就有了)

  • replace

傳入這個prop，會覆蓋history而不是新增history進去

< NavLink >

其實就是<Link>的加強版，主要是用在當前如果這個link是active的話，我們可以幫他增加一些樣式

< Switch >

讓只有第一個match到location的Route才會render出component

< Redirect >

顧名思義就是用來轉頁用的，而預設是會override當前在history stack的位置

< Prompt >

當user要轉到其他頁面時可以用來跳出提醒，這在有表單的頁面是很常用到的功能

< StaticRouter >

這個主要是server-side rendering時用到的，有時在test的時候也會使用到
